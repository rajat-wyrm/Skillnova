Task Validation Rules

All tasks submitted by interns go through a validation process to ensure quality standards are met.

1. Submission Requirements
- Complete all required fields before marking a task as "Done."
- Attach relevant files, code links, or screenshots as evidence of completion.
- Write a brief description of what was accomplished and any deviations from the original plan.

2. Validation Criteria
- Correctness: Does the output meet the task requirements and acceptance criteria?
- Code Quality: Is the code clean, well-documented, and following best practices?
- Testing: Were appropriate tests written and do they pass?
- Documentation: Are any necessary docs updated to reflect the changes?

3. Review Process
- Once submitted, your task enters "Review" status.
- Your mentor will review within 2 business days.
- Tasks may be approved, sent back for revision, or escalated for discussion.
- Feedback will be provided on the task card within the project board.

4. Common Rejection Reasons
- Incomplete implementation — missing edge cases or requirements.
- No test coverage — tasks without tests are sent back by default.
- Poor documentation — unclear commit messages or no description.
- Code style violations — not following the team's linting rules.

5. Resubmission
- Address all feedback comments before resubmitting.
- Update the task description with what was changed.
- Resubmitted tasks are prioritized in the review queue.
